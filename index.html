CSS: Positioning & Display

By default the browser will render elements from left to right, top to bottom.

CSS includes properties that change how a browser positions elements.

POSITION

Description
static
  • This is the default value.
  • Default width is the full width of the parent element.
  • Default position is on the left side.
  • Prevents other elements from appearing in the same horizontal space.
relative
  • Positions an element relative to its default position.
  • Used with offset properties.
  • Specify how the element should be moved, relative to its default position.
absolute
  • Used to displace an element from its static position, (top left of its container).
  • When offset properties are set, these will be relative to the top left corner of the page, not the container.
  • All other elements on the page will ignore the element and act like it is not present.
fixed
  • Fixes an element to a specific position on the page.
  • Used with offset properties.
  • Often used for navigation bars.
sticky
  • Scrolls with user but sticks to specified position.
  • Used with offset properties.

The element will remain in its relative position, and scroll as usual.

When it reaches the specified position it will stick to that position
until it reaches the bottom of its parent container where it
will “unstick” and rejoin the flow of the document.

z-index

When boxes have a combination of different positions they can overlap.

  • z-index controls how far back or forward an element should appear when elements overlap.
  • Accepts integer values, which define the order in which elements should be layered.
  • Does not work on static elements.


Offset Properties

Move From: Description
top Moves the element down from the top.
bottom Moves the element up from the bottom.
left Moves the element away from the left side (to the right).
right Moves the element away from the right side (to the left).


  • Values can be in pixels, ems, or percentages, amoung others.
  • Offset properties will not work if the position is default static.
  • Offsetting the relative element will not affect the positioning of other elements.

Display

Every HTML element has a default display value that dictates if it can share horizontal space with other elements.

Some elements fill the entire browser from left to right, regardless of the size of their content.

Other elements only take up as much horizontal space as their content requires and can be directly next to other elements.

display: inline

Inline is the default display for some elements, such as 'em', 'strong' or 'a' elements.

Inline elements have a box that wraps tightly around their content, only taking up the amount of space necessary to display their content
and not requiring a new line after each element.

These elements cannot be altered in size with the height or width CSS properties.

You can use this property to make any element an inline element.

display: block

Block-level elements are not displayed in the same line as the content around them, and fill the entire width of the page by default.

However, their width property can be set.

Unless otherwise specified, they are the height necessary to accommodate their content.

Elements that are block-level by default include all heading elements, paragraph, div and footer elements.

Inline-block elements can appear next to each other and we can specify their dimensions using the width and height properties.

float: right;

moves as far right as possible, in its container

float: left

Used to move an element as far left or as far right as possible in the container.

Commonly used for wrapping text around an image.


This works for static and relative positioned elements.

Floated elements must have a width specified. Otherwise, the element will assume the full width of its containing element.


(However, moving elements left or right for layout purposes is better suited for tools like CSS grid and flexbox.)

clear: left

The float property can also be used to float multiple elements at once.
However, when multiple floated elements have different heights, it can affect their layout on the page.
Specifically, elements can “bump” into each other and not allow other elements to properly move to the left or right.

The clear property specifies how elements should behave when they bump into each other on the page.

It can take on one of the following values:

left The left side of the element will not touch any other element within the same containing element.
right The right side of the element will not touch any other element within the same containing element.
both Neither side of the element will touch any other element within the same containing element.
none The element can touch either side.

For example, if all divs on a page are floated to the left side.
An element with class 'special' may not move all the way to the left because a taller div blocked its positioning.
Then by setting its clear property to left, the special div will be moved all the way to the left side of the page.
